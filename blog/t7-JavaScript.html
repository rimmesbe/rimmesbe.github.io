JavaScript vs. Ruby

February 2, 2015

Ruby and JavaScript are both programming languages. In this blog I will be going over some of their fundamental differences.

In Ruby we have a ‘hash’, which is a type of data structure object not very different than an array. Unlike an array hashes are not indexed by number variables, instead they are given keyword variables to point to their data.

Now in JavaScript there are no “hash” objects, there are only objects. JavaScript objects look very much like Ruby hashes. Objects hold data about the object in a similar key to value structure, but in the JavaScript case the ‘key’ is called a property. These properties are connected to values that are packaged with the object.

But don’t objects get their properties from the class they were spawned from? That is the case in Ruby, but in JavaScript there are no classes. If you want to pump out multiple copies of the same object you would create a function that spawns copies of that object with the properties you want. You could also just create an instance of a brand new object directly, hard coding its properties right there.

Another difference you may be noticing about JavaScript compared to Ruby is that JavaScript does not have premade iterators. Ruby has a plethora of packaged loops that serve all different operations along with documents listing what they are and how they work. In JavaScript you have to create your own loops. JavaScript comes with three types of loops; while, do-while, and for. You are responsible for creating your own iterator variables and local variables for holding values. Unfortunately there are no official JavaScript documents which list of all the different ways to use loops to manipulate data.
